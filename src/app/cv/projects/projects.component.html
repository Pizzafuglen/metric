• World of Trash (Zuul Framework)

    During our first semester we were tasked with producing a small game, based on the World of Zuul framework. The first part of the game was to be done as only a console game, and in the second we had to include a GUI using JavaFX.

  • CMS Module

    github.com/LangeHenrik/SDU-SE-F22

    During our second semester we were tasked with creating a CMS module, in a larger search system, for an imaginary electronics company. We implemented the backend in the beginning of our project, finishing off producing the UI. The entire project was conducted using the SCRUM framework.

  • Cyber-Physical System

    github.com/Pizzafuglen/E22-Sem-3-Brewing-Project-Java

    During our third semester we were tasked with creating a system that allowed for communication between an end-user and a physical device, using an HTTP-server as a digital means of transporting information. The project was finished using an agile, homebrew-SCRUM framework, with its own standalone UI finished in a separate project

  • Component Based System

    github.com/Pizzafuglen/SEM-F23

    During our fourth semester, we were tasked with implementing a system, allowing for control and manipulation of a simulation of the SDU I4.0-Lab. The implementation included component-oriented subsystems, communicating internally using HTTP-requests, a user-interface, and the use of 3 different protocols (HTTP, SOAP, and MQTT)

  • CoSELab Software Analysis

    source.coderefinery.org/coselab/sducodeanalysis

    As mentioned in the "about me"-section, I have spent a lot of time working on a project along one of my professors; the idea behind the project, is to allow developers insight into the code they write, beyond the standard shallow analysis available within most versioning tools (not excluding it, just adding to it). The project is written in PHP/Laravel, and is run both as a standalone local application, as cloud-based solution, and a containerized application.
